摘要
对官网博客 GNU Guix 1.1.0 released 的翻译

我们隆重宣布 GNU Guix 1.1.0 的发布!

本次发行包含:

  • ISO-9660 安装镜像
  • 虚拟机镜像
  • 提供源码包及二进制的 tar 压缩文件, 以在现有 GNU/Linux 发行版上安装 guix 包管理器


Guix 用户可以通过 guix pull 进行更新

如果您想知道安装 Guix System 是什么样的, 下面的视频概述了引导式安装的过程:

还有更多 入门视频

距离上个版本已过去 11 个月, 期间有 201 位贡献者提交了代码和软件包
本次更新包含超过 14000 次提交

以下是 工具 方面的亮点:

  • 新的 guix deploy 工具允许您一次部署多台机器, 无论是 通过 SSH 的远程机器 还是 虚拟专用服务器(VPS)上的机器
  • channel 的作者现在可为用户撰写新闻条目, 用户通过 guix pull --news 可轻松读取,
    如果您已经在使用 Guix, 那么您可能已经阅读了这些新闻!
  • 新的命令 guix system describe 可以得到部署系统时使用的 channel 的 commit, 并包含一个指向你系统配置文件的链接
    精确的来源跟踪提供了 "准确了解在两个不同的系统实例中发生的变化" 的能力, 该功能建立在新的 provenance 服务上
  • guix pack 现在改进了对 Singularity/Docker 的镜像支持(新增 --entry-point 选项)
  • 有一个新的 guix time-machine 命令, 这符合你的期望 :-), 而且它受益于 guix lint 改进了对 Software Heritage 的集成
  • guix challenge 现在新增了实时的 diff 对比功能, 这是近期可重现构造峰会(Reproducible Build Summit)的 成果
  • Guix 现在可以 publish/download lzip-compressed 的 substitutes, 这大大降低了带宽要求
  • guix system 现在支持 --target 选项, 为完整系统的交叉编译提供了一些支持, 更多信息请见该 FOSDEM talk
  • Guix 现在 运行于 Guile 3, 这大大提高了性能
  • The manual 现在包括一本 cookbook, 其中包含 The manual 中未覆盖主题的教程


以下是 发行版(distro) 方面的亮点:

  • 最大的变化是, 包的依赖关系图, 现在基于一组精简化的 "binary seed" (实现可完全审查 bootstrap 的关键步骤), 更多内容即将推出
  • Guix System 的图形安装程序 已经历许多错误修复和改进, 根据 1.0.0 中发现的 bug, 我们为其开发了 自动化测试框架
    CI 会针对不同配置自动测试(encrypted root, non-encrypted root, with or without a desktop environment)
  • 添加了 3514 个包, 总计超过 13000 个包, 升级了 3368 个软件包

该发行版附带了 GNU libc 2.29, GCC 9.3, GNOME 3.32, MATE 1.24.0, Xfce 4.14.0, Linux-libre 5.4.28, LibreOffice 6.4.2.2 等

  • 添加了 19 个新服务，特别新增了对运行 NFS servers, nftables tables, 甚至 high-level Web service, 如 Patchworkk
  • 构建系统增添了对 Node/Julia/Qt 的支持, 从而能更轻松地为这些生态编写包定义, 此外一个新的 copy-build-system 可执行您可能期望的工作


在 编程接口 与 底层架构, 许多事情也发生了变化, 尤其是:

  • 新的 with-build-handler 语法形式(form), 能够更好地支持 grafts 机制中引入的动态依赖处理
    在以后会有更多介绍, 但足以说明的是, 它修复了一个 长期存在的 user interface 和 performance issue
  • 模块 (guix remote) 下的 remote-eval 支持远程执行 作为 G-expression 的 Scheme 代码, 过程中会首先构建并部署该代码所依赖的所有组件
    此功能是实现 guix deploy(操控对象为远程主机) 与 guix system reconfigure(系统重配置) 之间代码共享的技术基石
    类似的, 新增的 eval/container 可在 "由系统自动完成资源配置" 的容器环境中执行代码
  • 新的 lower-gexp 返回 G-expression 的低级 IR, remote-eval, eval/container, gexp->derivation 均基于 lower-gexp 实现
  • with-parameters 语法形式(form), 允许你将特定对象(比如 package) 固定到特定的系统, 或交叉编译的目标上
  • 底层操作的执行效率得到 显著提升


这是个很长的列表! NEWS 文件 列出了更多您可能感兴趣的更改与您在意的错误修复
